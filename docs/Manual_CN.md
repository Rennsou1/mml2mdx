# mml2mdx MML 语法手册

> **mml2mdx** — MXDRV2 用 MML 编译器，兼容 NOTE v0.85  
> 将 MML（Music Macro Language）源文件编译为 MDX 格式

---

## MML 结构

### 文件头

MML 文件由 `#` 伪命令（指令）和通道 MML 数据组成。文件头通常包含标题和可选的 PCM 文件指定：

```
#title "曲名"                  ; 标题（必须）
#pcmfile "drums.pdx"           ; PDX 文件（使用 PCM 通道时指定）
```

一个完整的 MML 文件示例：

```
#title "demo"
#pcmfile "drums.pdx"

@1={                                       ; 乐器定义
 31,  0,  0, 15,  0,  0, 0, 1, 0, 0, 0,   ; OP1
 31,  0,  0, 15,  0,  0, 0, 1, 0, 0, 0,   ; OP2
 31,  0,  0, 15,  0,  0, 0, 1, 0, 0, 0,   ; OP3
 31,  0,  0, 15,  0,  0, 0, 1, 0, 0, 0,   ; OP4
  0,  7, 15                                ; CON, FL, OP-mask
}

A t120@1v15l4o4 cdefgab>c                  ; 通道 A
B      @1v12l4o3 efgab>cde                 ; 通道 B
```

### 乐器定义

FM 音色使用 `@N={...}` 语法定义，包含 **47 个数值**：

```
@<音色号>={
  AR, DR, SR, RR, SL, OL, KS, ML, DT1, DT2, AME,   ; OP1（M1）
  AR, DR, SR, RR, SL, OL, KS, ML, DT1, DT2, AME,   ; OP2（C1）
  AR, DR, SR, RR, SL, OL, KS, ML, DT1, DT2, AME,   ; OP3（M2）
  AR, DR, SR, RR, SL, OL, KS, ML, DT1, DT2, AME,   ; OP4（C2）
  CON, FL, OP
}
```

| 参数 | 说明 | 范围 |
|------|------|------|
| AR | Attack Rate | 0-31 |
| DR | Decay Rate | 0-31 |
| SR | Sustain Rate | 0-31 |
| RR | Release Rate | 0-15 |
| SL | Sustain Level | 0-15 |
| OL | Output Level (Total Level) | 0-127 |
| KS | Key Scaling | 0-3 |
| ML | Multiplier | 0-15 |
| DT1 | Detune 1 | 0-7 |
| DT2 | Detune 2 | 0-3 |
| AME | AM Enable | 0-1 |
| CON | Connection (Algorithm) | 0-7 |
| FL | Feedback Level | 0-7 |
| OP | Operator Mask | 0-15 (bit0-3=OP1-4) |

#### 音色宏

音色定义的 47 个参数之后可追加 MML 命令，切换到此音色时自动执行：

```
@5={
 ...47个参数...,
 MP1,4,20 MD5       ; 音色宏：切换到 @5 时自动执行 MP 和 MD
}
```

使用 `SMON` / `SMOF` 控制音色宏展开：

```
SMON                ; 启用音色宏展开（默认）
SMOF                ; 禁用音色宏展开
```

### 通道标识

| 通道 | 说明 |
|------|------|
| `A` ~ `H` | FM 通道 1-8（OPM 合成） |
| `P` ~ `W` | PCM/ADPCM 通道 1-8 |

- **多通道前缀**：`ABC` 表示 A、B、C 三个通道共享同一行 MML
- **通道分割**：在多通道行中用 `|...:...|` 为不同通道指定不同内容

```
ABC |cdef:efga:gab>c|4    ; A=cdef, B=efga, C=gab>c（共享音长 4）
```

### PCM 频率（F 值）

PCM 通道使用 `F` 命令设置采样率，完整 F 值表请参见 [PCM Setting.md](PCM%20Setting.md)。

#### 基本 ADPCM（无 #ex-pcm）

| F 值 | 采样率 |
|:----:|-------:|
| F0 | 3,906 Hz |
| F1 | 5,208 Hz |
| F2 | 7,812 Hz |
| F3 | 10,416 Hz |
| F4 | 15,625 Hz（默认） |

#### PCM8A 扩展（#ex-pcm）

| F 值 | 格式 | 采样率 |
|:----:|------|-------:|
| F5 | 16bit PCM | 15,625 Hz |
| F6 | 8bit PCM | 15,625 Hz |
| F7-F12 | 多种 | 20,833-31,250 Hz |

#### PCM8++ 扩展

F7-F41 支持多种格式（16bit/8bit/ADPCM）、采样率（15.6k-48k）、立体声、可变频率等。详见 [PCM Setting.md](PCM%20Setting.md)。

### 注释

```
; 行尾注释（分号后到行尾）
// 行尾注释（双斜杠后到行尾）
/* ... */ 块注释（仅在音色定义内部有效）
```

## 伪命令（# 指令）

所有 `#` 指令必须在行首出现（通道 MML 数据之前或独立一行）。

### 基本设置

| 指令 | 说明 |
|------|------|
| `#title "名称"` | 设置曲名（必须） |
| `#pcmfile "file.pdx"` | 指定 PCM 数据文件 |
| `#zenlen <值>` | 全音符 tick 数（默认 192） |
| `#octave-rev` | 反转 `<` `>` 方向 |

### 移调 / 音程

| 指令 | 说明 |
|------|------|
| `#tps <值>` | 全局移调（半音单位） |
| `#tps-all` | `#tps` 应用于全部通道（默认仅影响通道 A） |
| `#detune <值>` | 全局去调偏移量 |
| `#toneofs <值>` | 音色编号偏移 |
| `#flat "音符"` | 指定音符始终加降号，如 `#flat "be"` |
| `#sharp "音符"` | 指定音符始终加升号 |
| `#natural "音符"` | 取消 flat/sharp 指定 |
| `#normal "音符"` | 同 `#natural` |

### 和弦 / 效果控制

| 指令 | 说明 |
|------|------|
| `#coder` | 和弦不足时自动补休符（每侧至少 1 个） |
| `#ncoder` | 不自动补休符（默认） |
| `#glide [0\|1]` | Glide 加工条件（0=`&`后也加工, 1=仅首音） |
| `#reste [0\|1]` | 休符上的波形效果处理方式 |
| `#nreste` | 禁用休符波形效果 |
| `#cont [0\|1\|2]` | `&` 中波形相位继续控制 |
| `#wcmd [0\|1\|2]` | 波形效果中关联命令写入抑制 |
| `#wavemem` | 启用波形内存模式 |

### 编译选项

| 指令 | 说明 |
|------|------|
| `#noreturn` | 删除 MDX 中未使用的音色数据 |
| `#overwrite` | 允许音色重复定义（后定义覆盖前定义） |
| `#compress [0\|1]` | 音长压缩（0=仅休符, 1=含音符） |
| `#opt [flags]` | 最适化（`*`=全部, `dvqpt012`=选择性） |
| `#ex-pcm` | 启用 PCM8/PCM8++ 扩展频率模式 |

### 文件操作

| 指令 | 说明 |
|------|------|
| `#include "file.mml"` | 包含其他 MML 文件（递归支持） |
| `#load-tone "file"` | 加载二进制音色数据（覆盖现有定义） |
| `#load-wave "file"` | 加载波形数据 |
| `#save-tone ["file"]` | 导出音色数据（默认 tone.bin） |
| `#save-wave ["file"]` | 导出波形数据（默认 wave.bin） |

### 调试 / 工具

| 指令 | 说明 |
|------|------|
| `#list` | 显示 MML 列表 |
| `#nlist` | 显示音符解析列表 |
| `#ver [0\|1]` | 详细模式 |
| `#beep` | 出错时发出蜂鸣 |
| `#remove` | 出错时删除输出 MDX |
| `#pcmlist [0\|1]` | 输出 PCM 使用状况文件 |

## MML 命令

### 音符与休符

| 命令 | 说明 |
|------|------|
| `c` `d` `e` `f` `g` `a` `b` | 音符（C D E F G A B） |
| `c+` 或 `c#` | 升半音 |
| `c-` | 降半音 |
| `c=` | 还原记号 |
| `n<0-95>` | 按编号指定音符 |
| `r` | 休符 |

### 音长

| 命令 | 说明 |
|------|------|
| `l<1-64>` | 设置默认音长（后续省略音长的音符使用此值） |
| `c4` `c8` `c16` ... | 音符后跟数字指定音长（4=四分、8=八分...） |
| `c4.` | 附点（1.5 倍音长） |
| `c2..` | 复附点（1.75 倍音长） |
| `c%<tick>` | 按 tick 直接指定音长 |
| `c4&c4` | 连音（tie）：时值相加，不重新发音 |
| `c4&d4` | 跨音高连接（legato）：不断开 |

默认全音符 = 192 tick（可由 `#zenlen` 修改）

| 音长记法 | tick |
|----------|-----:|
| `1`（全音符） | 192 |
| `2`（二分） | 96 |
| `4`（四分） | 48 |
| `8`（八分） | 24 |
| `16` | 12 |
| `32` | 6 |

### 音量

| 命令 | 说明 |
|------|------|

### 波形效果

波形效果是 NOTE 的高级功能：通过预定义的 **数据序列**（波形内存），按节拍自动控制声像、音程、音量等参数，实现颤音、自动声像摇摆等效果。

工作原理：编译器在编译阶段将长音符拆分为多个短片段，每个片段之间自动插入对应的控制命令（p / D / v / y），从而模拟连续变化的效果。

#### 第一步：定义波形内存

波形内存是一个编号数据表，用 `*编号={...}` 定义：

```mml
*0={ 1, 0,   0, 10, 20, 30, 20, 10, 0, -10, -20, -30, -20, -10 }
;    │  │    └─ 后续全部为波形数据值（依次读取）
;    │  └─ 循环起始位置（0 = 从第一个数据开始循环）
;    └─ 是否循环（0=播放一次后停止，1=无限循环）
```

上例定义了一个 12 步的正弦波数据：`0→10→20→30→20→10→0→-10→-20→-30→-20→-10`，循环播放。

#### 第二步：将波形绑定到效果

| 效果 | 作用 | 实际插入的命令 |
|------|------|----------------|
| `AP` | 自动声像摇摆 | `p` 命令 |
| `DT` | 音程振动（颤音） | `D` 去调命令 |
| `TD` | 第二路音程振动（可与 DT 叠加） | `D` 去调命令 |
| `VM` | 音量包络 | `v` / `@v` 命令 |
| `MV` | 第二路音量包络（可与 VM 叠加） | `v` / `@v` 命令 |
| `KM` | 硬件 LFO 灵敏度变化（需先 MH 设定） | PMS/AMS 寄存器 |
| `TL` | OP 出力电平变化（精细音色控制） | OPM TL 寄存器 |
| `YA` | 任意寄存器写入 | `y` 命令 |

使用格式（以 AP 为例，其他效果格式完全相同）：

| 命令 | 说明 |
|------|------|
| `AP<波形号>,<步数>,<同步>` | 绑定波形并启动效果 |

- **波形号**：`*N` 定义的编号
- **步数**：每隔多少 tick 读取一次波形数据
- **同步模式**：`0`=不与音符同步（自由运行）、`1`=每次按键重新开始、`2`=不同步但 KeyOff 后继续

**实际效果示例** — 自动声像摇摆：

```mml
; 定义摇摆波形：左→中→右→中 循环
*0={ 1, 0,   1, 2, 3, 2, 1 }
;            p1  p2  p3  p2  p1（左→右→左 循环）

; 使用：每 12 tick 切换一次声像
A @1v15 AP0,12,1 l1 o4 cdef
; 编译后等效于：每 12 tick 自动插入 p1→p2→p3→p2→p1...
```

#### 效果子命令

每种效果都有配套的子命令（将 `XX` 替换为 AP/DT/TD/VM/MV/KM/TL/YA）：

| 命令 | 说明 |
|------|------|
| `XXD<值>` | 延迟：效果在第 N tick 后才开始（负值=从 KeyOff 后起算） |
| `XXS<值>` | 倍率：波形数据乘以此值（放大/缩小效果强度） |
| `XXL<值>` | 步长变更：运行中修改读取间隔 |
| `XXON` | 恢复之前停止的效果 |
| `XXOF` | 停止效果 |

#### TL 专属子命令

| 命令 | 说明 |
|------|------|
| `TLM<0-15>` | 指定受控 OP 掩码（bit0-3=OP1-4，-1=自动选调制 OP） |
| `TLT<音色号>` | 设置参考音色号（用于计算调制 OP，不切换音色） |

### 音域自动切换

|------|------|
| `KS<0-7>` | 按音域自动切换音色 |
